# 家庭药品管理系统 - 常见问题排查指南

## 一、后端问题

### 1.1 项目无法启动

#### 问题现象
```
Error: Unable to start embedded Tomcat
```

#### 排查步骤
1. **检查端口占用**
   ```bash
   # Windows
   netstat -ano | findstr :8080
   
   # Linux/Mac
   lsof -i :8080
   ```
   
   **解决方案**：修改 `application.yml` 中的端口配置：
   ```yaml
   server:
     port: 8081  # 修改为其他端口
   ```

2. **检查 JDK 版本**
   ```bash
   java -version
   ```
   确保版本为 17 或更高。

3. **检查配置文件**
   - 确认 `application.yml` 格式正确（注意缩进）
   - 确认数据库连接信息正确

---

### 1.2 数据库连接失败

#### 问题现象
```
com.mysql.cj.jdbc.exceptions.CommunicationsException: Communications link failure
```

#### 排查步骤
1. **检查 MySQL 服务状态**
   ```bash
   # Windows
   net start mysql
   
   # Linux
   systemctl status mysql
   
   # Mac
   brew services list | grep mysql
   ```

2. **检查连接配置**
   ```yaml
   spring:
     datasource:
       url: jdbc:mysql://localhost:3306/medi_home?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
       username: root          # 确认用户名正确
       password: your_password # 确认密码正确
   ```

3. **检查数据库是否存在**
   ```sql
   SHOW DATABASES LIKE 'medi_home';
   ```

4. **检查用户权限**
   ```sql
   GRANT ALL PRIVILEGES ON medi_home.* TO 'your_user'@'localhost';
   FLUSH PRIVILEGES;
   ```

---

### 1.3 MyBatis 报错

#### 问题现象
```
Invalid bound statement (not found): com.medihome.mapper.xxxMapper.xxxMethod
```

#### 原因与解决
1. **Mapper XML 文件未编译**
   - 检查 `pom.xml` 中是否正确配置了资源目录
   - 执行 `mvn clean compile` 重新编译

2. **Mapper 接口与 XML 命名空间不匹配**
   - 确认 XML 中的 `namespace` 与 Mapper 接口全路径一致

3. **方法名不匹配**
   - 确认 XML 中的 `id` 与 Mapper 接口方法名一致

---

### 1.4 JWT 验证失败

#### 问题现象
```
HTTP 401 Unauthorized
```

#### 排查步骤
1. **检查请求头**
   ```
   Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
   ```
   确保格式正确，注意 `Bearer` 后有空格。

2. **检查 Token 是否过期**
   - Token 默认 7 天过期
   - 重新登录获取新 Token

3. **检查 JWT 密钥**
   - 确保前后端密钥一致
   - 修改密钥后需要重新登录

---

### 1.5 文件上传失败

#### 问题现象
```
File upload failed
```

#### 排查步骤
1. **检查上传目录权限**
   ```bash
   # Linux/Mac
   chmod 755 ./uploads
   
   # 确保目录存在
   mkdir -p ./uploads/drugs
   ```

2. **检查文件大小限制**
   ```yaml
   spring:
     servlet:
       multipart:
         max-file-size: 10MB
         max-request-size: 100MB
   ```

3. **检查磁盘空间**
   ```bash
   df -h
   ```

## 二、前端问题

### 2.1 npm install 失败

#### 问题现象
```
npm ERR! code EACCES
npm ERR! syscall access
```

#### 解决方案
1. **清除缓存**
   ```bash
   npm cache clean --force
   rm -rf node_modules
   rm package-lock.json
   npm install
   ```

2. **使用淘宝镜像**
   ```bash
   npm config set registry https://registry.npmmirror.com
   npm install
   ```

3. **使用 yarn**
   ```bash
   yarn install
   ```

---

### 2.2 前端无法访问后端 API

#### 问题现象
```
Proxy error: Could not proxy request /api/xxx from localhost:3000 to http://localhost:8080
```

#### 排查步骤
1. **检查后端服务是否启动**
   ```bash
   curl http://localhost:8080/api/user/login
   ```

2. **检查代理配置**
   ```javascript
   // vue.config.js
   devServer: {
     proxy: {
       '/api': {
         target: 'http://localhost:8080',  // 确认后端地址正确
         changeOrigin: true
       }
     }
   }
   ```

3. **检查跨域配置**
   - 确保后端 `WebConfig.java` 中 CORS 配置正确
   - 确认 `allowedOriginPatterns` 包含前端地址

---

### 2.3 页面白屏

#### 问题现象
页面加载后显示空白，控制台无报错或报错信息不明确。

#### 排查步骤
1. **检查路由配置**
   - 确认路由路径正确
   - 检查是否有路由守卫阻止访问

2. **检查组件导入**
   ```javascript
   // 确保组件正确导入
   import Dashboard from '@/views/Dashboard.vue'
   ```

3. **检查 Pinia Store**
   - 确认 Store 已正确注册
   - 检查是否有循环依赖

---

### 2.4 Element Plus 图标不显示

#### 问题现象
图标显示为方框或空白。

#### 解决方案
```javascript
// main.js
import * as ElementPlusIconsVue from '@element-plus/icons-vue'

for (const [key, component] of Object.entries(ElementPlusIconsVue)) {
  app.component(key, component)
}
```

## 三、数据库问题

### 3.1 中文乱码

#### 问题现象
数据库中存储的中文显示为乱码。

#### 解决方案
1. **检查数据库字符集**
   ```sql
   SHOW VARIABLES LIKE 'character_set%';
   ```

2. **修改数据库字符集**
   ```sql
   ALTER DATABASE medi_home CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

3. **检查连接字符集**
   ```yaml
   url: jdbc:mysql://localhost:3306/medi_home?useUnicode=true&characterEncoding=utf-8
   ```

---

### 3.2 数据插入失败

#### 问题现象
```
Data truncation: Data too long for column 'xxx'
```

#### 解决方案
1. **检查字段长度**
   - 修改字段长度：`ALTER TABLE xxx MODIFY COLUMN xxx VARCHAR(500);`

2. **检查数据格式**
   - 日期格式：`yyyy-MM-dd`
   - 日期时间格式：`yyyy-MM-dd HH:mm:ss`

---

## 四、性能问题

### 4.1 查询速度慢

#### 优化建议
1. **添加索引**
   ```sql
   CREATE INDEX idx_drug_user_status ON drug(user_id, status);
   ```

2. **分页优化**
   - 避免深分页（pageNum > 1000）
   - 使用游标分页或搜索条件限制

3. **SQL 优化**
   - 避免 SELECT *
   - 使用 EXPLAIN 分析执行计划

---

### 4.2 内存占用高

#### 排查步骤
1. **检查 JVM 参数**
   ```bash
   java -Xms512m -Xmx2g -jar app.jar
   ```

2. **检查是否有内存泄漏**
   - 使用 VisualVM 或 Arthas 分析

---

## 五、安全相关问题

### 5.1 密码加密问题

#### 问题现象
用户无法登录，提示密码错误。

#### 排查步骤
1. **确认密码加密方式**
   - 使用 BCrypt 加密
   - 检查加密后的密码格式（以 `$2a$` 开头）

2. **检查加密配置**
   ```java
   @Bean
   public PasswordEncoder passwordEncoder() {
       return new BCryptPasswordEncoder();
   }
   ```

---

### 5.2 JWT 安全问题

#### 安全建议
1. **定期更换密钥**
   ```yaml
   jwt:
     secret: ${JWT_SECRET:default-secret}  # 从环境变量读取
   ```

2. **设置合理的过期时间**
   - 访问令牌：7天
   - 刷新令牌：30天

3. **使用 HTTPS**
   - 生产环境必须启用 HTTPS
