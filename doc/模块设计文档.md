# 家庭药品管理系统 - 模块设计文档

## 1. 系统架构概述

### 1.1 技术栈
- **前端**: Vue3 + Element Plus + Axios
- **后端**: Spring Boot 3 + MyBatis-Plus + Spring Security + JWT
- **数据库**: MySQL 5.7
- **JDK**: 17+

### 1.2 模块划分
```
medi-home/
├── backend/                    # 后端项目
│   ├── src/main/java/
│   │   └── com/medihome/
│   │       ├── controller/     # 控制层
│   │       ├── service/        # 业务层
│   │       ├── mapper/         # 数据访问层
│   │       ├── entity/         # 实体类
│   │       ├── dto/            # 数据传输对象
│   │       ├── vo/             # 视图对象
│   │       ├── config/         # 配置类
│   │       ├── security/       # 安全相关
│   │       └── utils/          # 工具类
│   └── src/main/resources/
│       └── mapper/             # MyBatis XML
├── frontend/                   # 前端项目
│   ├── src/
│   │   ├── api/                # API接口
│   │   ├── views/              # 页面组件
│   │   ├── components/         # 公共组件
│   │   ├── router/             # 路由配置
│   │   ├── store/              # 状态管理
│   │   └── utils/              # 工具函数
│   └── public/
└── doc/                        # 文档
```

## 3. 接口设计

### 3.1 统一响应格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 3.2 通用状态码
| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录/Token失效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 3.3 用户模块接口

#### 3.3.1 用户注册
- **接口**: `POST /api/user/register`
- **请求体**:
```json
{
  "phone": "13800138000",
  "password": "123456",
  "verifyCode": "123456"
}
```
- **响应**:
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": 1,
      "phone": "13800138000",
      "nickname": null,
      "role": 1
    }
  }
}
```

#### 3.3.2 用户登录
- **接口**: `POST /api/user/login`
- **请求体**:
```json
{
  "phone": "13800138000",
  "password": "123456"
}
```
- **响应**: 同注册接口

#### 3.3.3 获取验证码
- **接口**: `GET /api/user/verify-code?phone=13800138000`
- **响应**:
```json
{
  "code": 200,
  "message": "验证码已发送",
  "data": null
}
```

#### 3.3.4 获取当前用户信息
- **接口**: `GET /api/user/info`
- **请求头**: `Authorization: Bearer {token}`
- **响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "phone": "13800138000",
    "nickname": "张三",
    "avatar": null,
    "role": 1
  }
}
```

#### 3.3.5 修改用户信息
- **接口**: `PUT /api/user/info`
- **请求体**:
```json
{
  "nickname": "张三",
  "avatar": "http://xxx/avatar.jpg"
}
```

#### 3.3.6 修改密码
- **接口**: `PUT /api/user/password`
- **请求体**:
```json
{
  "oldPassword": "123456",
  "newPassword": "654321"
}
```

## 5. 核心业务逻辑

### 5.1 药品过期检查
- **触发时机**: 
  - 每日定时任务（凌晨2点）
  - 用户登录时
  - 查看药品列表时
- **逻辑**: 
  - 计算距离过期天数
  - 小于等于提醒天数：标记为"即将过期"，生成消息
  - 已过期：标记为"已过期"，生成消息

### 5.2 库存预警
- **触发时机**:
  - 用药记录添加时自动扣减库存
  - 手动调整库存时
- **逻辑**:
  - 库存 <= 预警阈值：生成库存不足消息

### 5.3 用药提醒检查
- **触发时机**: 定时任务（每分钟）
- **逻辑**:
  - 检查当前时间是否匹配提醒时间
  - 检查频率类型（每日/每周/自定义）
  - 生成待办消息
