# 家庭药品管理系统

1. 文档概述

1.1 文档目的

本文档用于明确家庭药品管理系统的核心需求、功能范围、交互逻辑及非功能要求，作为系统设计、开发、测试及验收的核心依据，确保开发成果符合家庭用户对药品管理的实际需求，解决家庭药品杂乱、过期遗忘、用法模糊等痛点。

1.2 项目背景

当前家庭普遍存在药品存放混乱、过期药品未及时清理、药品用法用量记忆模糊、家庭成员用药记录不清晰等问题，易导致用药安全隐患（如误食过期药、用量不当）、药品浪费（过期未使用）。为解决上述问题，开发一款简单、易用、高效的家庭药品管理系统，实现药品全生命周期管理，保障家庭成员用药安全，减少药品浪费。

1.3 适用范围

本文档适用于家庭药品管理系统的开发人员、测试人员、产品人员及使用该系统的家庭用户，明确各角色的需求边界与系统功能标准。

1.4 术语定义

- 药品信息：包含药品名称、规格、生产厂家、有效期、用法用量、适应症、禁忌、储存条件等核心信息；

- 过期提醒：系统根据药品有效期，提前设定时间（如7天、30天）向用户推送过期预警；

- 分类管理：按药品类型（感冒药、退烧药、消炎药等）、使用人群（成人、儿童、老人）等维度对药品进行分类；

- 用药记录：记录家庭成员每次用药的药品、用量、时间及用药人信息；

- 家庭成员管理：添加、编辑家庭成员信息，区分不同成员的用药权限与记录。

2. 总体需求

2.1 核心目标

1. 实现家庭药品的集中录入、分类管理，解决药品存放混乱问题；

2. 提供药品过期提醒功能，减少过期药品浪费，避免误食过期药；

3. 记录药品用法用量、适应症等信息，方便用户快速查询，保障用药安全；

4. 记录家庭成员用药情况，便于追溯用药历史，适配不同成员的用药需求；

5. 操作简单、界面简洁，适配老人、年轻人等不同年龄段用户使用；

6. 保障数据安全，支持药品信息、用药记录的备份与恢复，防止数据丢失。

2.2 用户画像

- 核心用户：家庭主妇、家中长辈（负责药品采购与管理）；

- 次要用户：家庭其他成员（查询药品信息、记录自身用药情况）；

- 用户需求特点：操作便捷、功能直观，无需复杂学习，支持快速查询与录入。

2.3 系统运行环境

2.3.1 移动端（优先支持）

- 系统版本：Android 8.0及以上、iOS 12.0及以上；

- 硬件要求：内存≥2GB，存储空间≥100MB；

- 网络要求：支持离线使用（核心功能），联网状态下支持数据备份、云端同步。

2.3.2 网页端（可选支持）

- 浏览器：Chrome、Edge、Firefox等主流浏览器（最新版本）；

- 硬件要求：CPU≥1GHz，内存≥4GB，存储空间≥50MB；

- 网络要求：联网状态下使用，支持数据与移动端同步。

3. 功能需求

3.1 基础功能模块

3.1.1 用户注册与登录

- 支持手机号注册，获取验证码完成验证，设置登录密码；

- 支持密码登录、验证码快捷登录；

- 支持“忘记密码”功能，通过手机号找回密码；

- 登录后默认进入系统首页，未登录状态下仅可查看系统简介，无法使用核心功能。

3.1.2 家庭成员管理

- 支持添加家庭成员，录入姓名、年龄、性别、过敏史、常用药品等信息；

- 支持编辑、删除家庭成员信息（仅管理员/创建者可操作）；

- 支持设置家庭成员权限（如管理员、普通成员，管理员可修改系统设置、添加/删除药品）；

- 支持快速切换当前用药人，方便记录不同成员的用药情况。

3.2 核心功能模块（药品管理）

3.2.1 药品录入

- 支持手动录入药品信息：药品名称、规格、生产厂家、生产日期、有效期、用法用量、适应症、禁忌、储存条件、采购日期、采购数量、存放位置；

- 支持扫码录入（扫描药品包装上的条形码/二维码），自动识别药品核心信息（名称、规格、厂家、有效期），减少手动录入工作量；

- 支持批量录入药品，可导入Excel表格（提供模板），批量添加多个药品；

- 录入时自动校验有效期（若已过期，提示用户并标记为过期药品）；

- 支持上传药品图片（如药品包装、说明书），方便后续查看。

3.2.2 药品分类管理

- 支持默认分类：感冒药、退烧药、消炎药、肠胃药、外用药、维生素、其他，用户可直接选择分类；

- 支持自定义分类，用户可添加、编辑、删除分类（如“儿童专用药”“老人专用药”）；

- 支持按分类筛选药品，快速查找对应类型的药品；

- 支持按存放位置筛选药品（如“客厅药箱”“卧室抽屉”），方便快速定位药品。

3.2.3 药品查询

- 支持模糊查询：输入药品名称、厂家关键词，快速检索相关药品；

- 支持多条件筛选：按分类、有效期（未过期、即将过期、已过期）、使用人群、存放位置筛选；

- 点击药品可查看详细信息（所有录入的药品信息、图片、用药记录）；

- 支持药品收藏，常用药品可添加到收藏夹，快速访问。

3.2.4 药品过期提醒

- 支持自定义提醒时间：用户可设置提前7天、15天、30天提醒药品过期；

- 提醒方式：系统弹窗提醒、消息通知（移动端）、短信提醒（可选）；

- 首页显示“即将过期药品”“已过期药品”入口，方便用户快速处理；

- 已过期药品自动标记为红色，提醒用户及时清理；

- 支持批量清理已过期药品，清理后可选择删除或归档（归档后可查看历史记录）。

3.2.5 药品库存管理

- 支持修改药品库存数量（如使用后减少库存、补充药品后增加库存）；

- 支持设置库存预警，当药品库存低于设定值（如剩余1片/1盒），推送库存不足提醒；

- 记录药品库存变动历史（如“2026-03-10，使用1片，库存剩余5片”）；

- 支持批量盘点库存，核对实际库存与系统记录，一键调整库存数量。

3.3 核心功能模块（用药管理）

3.3.1 用药记录

- 支持添加用药记录：选择用药人、药品、用药时间、用药剂量、用药备注（如“饭后服用”“遵医嘱”）；

- 支持快速添加用药记录（选择常用药品、常用剂量，一键提交）；

- 支持按用药人、用药时间、药品筛选用药记录；

- 用药记录自动关联药品信息，可查看该药品的用法用量、禁忌等信息；

- 支持编辑、删除用药记录，仅管理员或用药记录添加者可操作。

3.3.2 用药提醒

- 支持设置用药提醒：选择用药人、药品、用药时间、用药频率（如“每日3次，每次1片”“每周1次”）；

- 提醒方式：系统弹窗、消息通知（移动端）、铃声提醒，支持设置提醒铃声；

- 提醒后可标记“已用药”，未标记则间隔一定时间（如15分钟）再次提醒；

- 支持暂停、恢复、删除用药提醒，可编辑提醒信息（如调整用药时间、频率）。

3.3.3 用药历史查询

- 支持按用药人、时间范围（今日、本周、本月、自定义）查询用药历史；

- 支持导出用药历史记录（Excel格式），方便备份或就医时提供；

- 显示用药统计（如某成员本周用药次数、常用药品）。

3.4 辅助功能模块

3.4.1 数据备份与恢复

- 支持手动备份：将药品信息、用药记录、家庭成员信息备份到本地（移动端）或云端；

- 支持自动备份：设置备份频率（如每日、每周），自动备份数据；

- 支持数据恢复：当数据丢失时，可通过本地备份或云端备份恢复数据；

- 支持数据同步：移动端与网页端数据同步，多设备登录可查看相同信息。

3.4.2 药品说明书查询

- 支持手动输入药品名称，查询药品详细说明书（用法用量、适应症、禁忌、不良反应等）；

- 支持扫码查询说明书（扫描药品包装条形码/二维码）；

- 支持收藏常用药品说明书，快速查看。

3.4.3 系统设置

- 支持修改登录密码、绑定手机号；

- 支持设置过期提醒时间、库存预警值；

- 支持设置消息通知权限（开启/关闭弹窗、铃声提醒）；

- 支持清除缓存、版本更新、关于系统等功能；

- 支持切换系统语言（中文简体、中文繁体）、主题（浅色、深色）。

4. 非功能需求

4.1 性能需求

- 响应速度：药品录入、查询、筛选等操作响应时间≤1秒；

- 稳定性：系统连续运行无崩溃，无异常报错，支持多用户同时操作（家庭场景≤5人）；

- 存储能力：支持至少1000条药品信息、5000条用药记录的存储，占用存储空间≤500MB；

- 离线能力：核心功能（药品查询、用药记录、过期提醒）支持离线使用，联网后自动同步数据。

4.2 易用性需求

- 界面简洁直观，操作流程简单，无需复杂学习，老人可轻松上手；

- 按钮、文字大小适中，移动端适配不同屏幕尺寸，操作便捷；

- 关键操作（如药品录入、用药提醒设置）提供引导提示；

- 支持扫码功能，减少手动输入，提升操作效率。

4.3 安全性需求

- 用户数据加密存储，密码采用加密算法，防止数据泄露；

- 支持账号退出功能，未登录状态下无法访问核心数据；

- 数据备份支持密码保护，防止备份文件被非法访问；

- 避免误操作：删除药品、用药记录等关键数据时，提供二次确认提示。

4.4 可扩展性需求

- 支持后续新增功能（如药品采购链接、医生咨询、用药知识科普）；

- 支持适配新的移动端系统版本、浏览器版本；

- 支持自定义字段（如新增药品“购买渠道”“价格”字段）。

4.5 兼容性需求

- 移动端：兼容Android 8.0及以上、iOS 12.0及以上的主流手机型号；

- 网页端：兼容Chrome、Edge、Firefox等主流浏览器最新版本；

- 数据兼容：支持Excel表格导入/导出，兼容常见的Excel格式（.xlsx、.xls）。

5. 界面需求

5.1 整体风格

简洁、清新、易用，以浅色为主色调（搭配蓝色辅助色），避免复杂装饰，突出核心功能；文字清晰、按钮明显，适配不同年龄段用户；移动端界面采用底部导航栏，方便单手操作。

5.2 核心界面布局

- 首页：显示即将过期药品、今日用药提醒、常用药品入口、家庭成员切换入口；

- 药品管理页：分类列表、药品搜索框、添加药品按钮、筛选条件；

- 用药管理页：用药记录列表、添加用药记录按钮、用药提醒设置入口；

- 家庭成员页：家庭成员列表、添加成员按钮、编辑/删除操作入口；

- 我的页：个人信息、系统设置、数据备份、说明书查询、关于系统等入口。

6. 验收标准

6.1 功能验收

- 所有功能模块按需求文档实现，无缺失功能；

- 操作流程顺畅，无卡顿、报错，响应时间符合性能需求；

- 数据录入、查询、修改、删除准确无误，数据同步正常；

- 提醒功能正常（过期提醒、用药提醒、库存提醒），提醒方式符合需求。

6.2 非功能验收

- 稳定性：连续运行24小时无崩溃、无异常；

- 易用性：老人（60岁以上）可在5分钟内学会核心操作（药品查询、添加用药记录）；

- 安全性：数据加密存储，未登录无法访问核心数据，关键操作有二次确认；

- 兼容性：在指定的移动端、浏览器上正常运行，无界面错乱、功能异常。
